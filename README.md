# 中证1000 时序择时(v4 定稿)

遗传规划(GP)公式化因子挖掘 + walk-forward 时序择时。**单标的指数多空择时,非选股策略。**

**成绩(2018–2026 回测,双边万5):年化 ~27% / 夏普 ~1.3 / 最大回撤 −17.8% / 卡玛 ~1.5**;同期持有 512100:4% / 0.19 / −46.3%。

---

## 策略流程速览

```
6指数日频OHLCV ──> GP逐季挖因子(rank IC) ──> 因子信号化(40日rank) ──> cos IC加权合成
                                                                        │
实盘/回测仓位 <── tanh(Φ⁻¹(π)) 映射 <── 组合信号40日分位 π <── (+熊态增强ΔS) ──┘
```

### 1. 数据

| 项 | 内容 |
|---|---|
| 行情 | 6 个 A 股宽基指数(沪深300/中证500/**中证1000**/创业板指/科创50/中证红利)日频 **开、高、低、收、成交量、成交额** |

### 2. 因子挖掘

**因子 = 表达式树**(公式化因子,Kakushadze 101 Alphas 范式),GP 每季度进化搜索:26 个算子(四则、`ts_std/ts_mean/ts_rank/ts_corr/ema/delta/ts_er/ts_argmax` 等)× 9 个叶子 × 窗口 {3,5,10,20,40,60,120,244}。种群 200、20 代、锦标赛选择、子树交叉 0.7 / 变异 0.25、深度≤6、复杂度罚 0.001。

**统计指标(挖掘与治理全流程):**

| 指标 | 用途 | 阈值 |
|---|---|---|
| rank IC(Spearman) | GP 适应度 + 名人堂门槛 | train/valid 均 ≥0.02 |
| 月度混合 rank IC | 入库门槛(validation 段) | ≥0.015 |
| 残差 IC(对现役库回归后) | 防同信息换马甲 | ≥0.01 |
| 相关筛 | 库内去重 | ≤0.9 |
| trailing 12月 IC | 逐季体检:黄牌/退役 | <0 黄牌,连续两季退役;<−0.02 立即退役 |
| cos IC | **持仓层加权**(见 §3) | >0 才有权重 |

**X / Y 窗口:**

| | 定义 | 窗口 |
|---|---|---|
| Y(预测目标) | 次日收益 `close→close`,horizon = **1 天** | — |
| X(因子回看) | 表达式内滚动窗 | 3~244 日(最长一年) |
| 信号化 | 因子值在自身历史的滚动百分位 ×2−1 | **40 日**(挖矿记账层用 120 日) |
| 权重估计 | trailing cos IC(季度末锁定,窗口末剔 1 日防泄漏) | **12 个月** |
| GP 训练/验证 | train / validation 切分 | 回看 10 年:前 8 年训练,近 2 年验证 |

**标的池与时间段:**
- "股票池" = **6 个宽基指数**(两级库:6 指数联合面板挖"类库" + 每指数单独挖"专属库");最终交易信号只用「类库 ∪ 专属中证1000」,执行标的单一(512100)。**不持有个股。**
- 时间轴:2013 起预热(元老记账)→ **2015 起建库**(逐季 walk-forward 重挖)→ **2018 起计成绩**。净值上每一天都是模型未见过的数据。

**指标时间序列:有,全程落盘。** 每因子逐日信号序列、逐季权重、组合信号 comb、映射分位 π、逐日仓位与收益,均存于 `results/ops26/`(挖矿存档,91 万行)与 `data/archive/`(`分资产逐日.csv`、`映射分位.csv`)、`results/final/逐日.csv`。

### 3. 怎么组合信号

1. **单因子信号化**:因子原始值 → 自身 40 日滚动百分位 ×2−1 → S_i ∈ [−1,1](消因子间量纲,秩变换对极端值免疫)
2. **cos IC 定权**:每季度末,把每个因子信号单独过一遍仓位映射得到 p_i,算过去 12 个月的 `cosIC_i = Σ(p_i·r_{t+1}) / √(Σp_i²·Σr²)` ——**分子就是该因子单独交易的盈亏**;权重 = max(cosIC, 0),黄牌减半,季内锁定
3. **凸组合**:`comb = Σ wᵢSᵢ / Σwᵢ`(88 个在役因子,人工元老已剔除);可用因子 <8 时整体减半(烧机保护)
4. **熊态增强**:iVIX/利率驱动的熊市概率触发家族再倾斜,产生小修正 ΔS 叠加(贡献约 +0.8pp,非熊态为 0)

> 为什么挖矿用 rank IC、加权用 cos IC:挖掘要**抗噪的方向筛选**(秩统计不被极端日骗),加权要**对齐盈亏的计量**(cos 分子即钱)。同一把尺子用两遍:cos挖+cos权过拟合(实测夏普 0.84),rank权丢强度信息(1.01);分工使用 = 1.37。

### 4. 怎么构建到策略(持仓设计)

**不是个股组合——持仓就是单一 ETF 的一个连续仓位数:**

```
π_t   = (comb+ΔS) 在同质参照系下的 40 日滚动百分位      ← 今天的综合信号,相对近两个月算高算低
仓位_t = tanh(Φ⁻¹(π_t)) ∈ (−1, 1)                      ← 平常小仓位晃、极端果断上满仓
```

- **多空口径**:仓位 ∈ [−1,1],负值做空(IM 股指期货实现空头腿);**纯多口径**:clip 到 [0,1],看空即平仓吃货基
- 信号 T 日收盘生成,**T+1 生效**;调仓付双边万5;未上仓位资金计货基收益
- 同质参照系:每季用本季权重回算窗口内 comb 作排位参照,避免窗口内混用新旧权重体制
- 实测均仓位约 60%,日换手约 30%

### 5. 无前视与防过拟合

三段数据(train/validation/成绩段完全隔离)· 逐季 walk-forward(只用当时可见数据)· 所有滚动窗口只回看 · 权重窗口末端剔 1 日(决策时点不可得的收益不进估计)· 入库三关 + 黄牌退役制 · 阈值预注册跑后不改 · 任何存档重放须先逐日复现引擎原值(<1e-12)。
