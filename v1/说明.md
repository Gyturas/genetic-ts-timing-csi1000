# 中证1000 时序择时 —— v1(rank IC 挖矿版)

这一版是**第一版定稿**:遗传规划用 **rank IC** 打分挖因子,再比较五种加权方式。
整包自含代码、行情、因子库存档、结果图表,可从零复现。

> 为什么单独冻结成 v1:下一版(v2)打算把挖矿打分换成 **cos IC**,挖矿代码不通用,
> 所以每一版各自带一份完整的挖矿脚本和产物,互不覆盖。

## 一句话结论

- **正典(线上口径)= 年化 20.7% / 夏普 1.01 / 回撤 −20.9% / 卡玛 0.99**(熊增强·多空,2018 起)
- 五种加权里 **cos IC 加权最好:24.7% / 1.20 / −18.7% / 1.32**,收益更高、回撤更小、仓位更省
- 详见 `结果/加权对比/`

## 五种加权方式(因子库同一批,只改怎么合成)

| 加权方式 | 做法 | 年化 | 夏普 | 回撤 | 卡玛 |
|---|---|---|---|---|---|
| 正典 | 剔除 IC≤0 的因子 + 按 rank IC 大小分配权重 | 20.7% | 1.01 | −20.9% | 0.99 |
| 截正等权 | 剔除 IC≤0 + 剩下的等权 | 21.0% | 1.02 | −20.9% | 1.00 |
| 等权 | 不剔除 + 全部等权(黄牌减半) | 21.3% | 1.04 | −20.9% | 1.02 |
| 纯等权 | 不剔除 + 全部等权(黄牌也不减半) | 21.5% | 1.04 | −20.4% | 1.05 |
| **cos IC** | 按 cos IC 加权(既看方向对错、也看信号强弱) | **24.7%** | **1.20** | **−18.7%** | **1.32** |

**为什么 cos IC 更好**:rank IC 只看"信号排序和涨跌排序对不对得上",把信号的绝对强弱
扔了;而时序择时赚的钱恰恰由仓位的方向和大小决定。cos IC 的分子 `Σ 仓位×收益` 就是
这个因子单独交易能赚多少钱,直接对齐真实盈亏。等权 vs 正典几乎打平(rank IC 权重里
约 3/4 是估计噪音,收缩成等权只微弱变好),真正的提升来自换掉 rank IC 这把尺子本身。

## 从零复现

```bash
pip install -r requirements.txt          # numpy pandas scipy matplotlib scikit-learn arch
export PYTHONPATH=$PWD

# 1. 挖因子(约1小时;加 --end 2015-06-30 可几分钟试跑)。产物落 结果_26算子/
python -m 引擎.挖矿_26算子

# 2. 五档加权对比图与表(读存档重算,不重挖,秒级)
python -m 引擎.加权对比

# 3. 单看正典口径回测(复现 20.7%/夏普1.01)
python -m 引擎.策略
```

挖矿只写 `结果_26算子/`,不覆盖 `结果/` 和 `数据/`。
`结果_26算子/因子逐日信号.csv.gz` 已随包附带,所以第 2、3 步无需先挖矿即可直接跑。
